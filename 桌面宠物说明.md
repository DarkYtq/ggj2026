# CatWidget 桌面宠物 —— 打包与使用说明

把 CatWidget 场景打包成独立程序后，它会变成一个**铺满屏幕、背景透明、无边框、置顶**的桌面宠物：
默认鼠标点击穿透到后面的应用，只有指针落在**小猫 / 信箱 / 弹窗 / 按钮**上时才接收点击；
可以**拖拽小猫**在屏幕上移动它，单击冒爱心、双击进关卡的原有交互保持不变。

> 注意：透明窗口效果**只在打包后的独立程序里生效**，在编辑器 Play 模式下看不到透明/穿透（编辑器里所有原生调用为空操作，仅方便调试逻辑）。

## 推荐：用打包脚本一键出包

打包脚本会在打包前**强制写好所有保证透明的设置**，避免手动漏项导致黑底。

- Unity 编辑器里：菜单 `Build > Desktop Pet - Windows x64` / `- macOS` / `- Both`，产物在项目根的 `Builds/`。
- 命令行（macOS/Linux）：
  ```bash
  ./build_desktop_pet.sh            # 两个平台
  ./build_desktop_pet.sh win        # 只打 Windows
  ./build_desktop_pet.sh mac        # 只打 macOS
  # 若脚本找不到 Unity，用环境变量指定：
  UNITY_PATH='/Applications/Unity/Hub/Editor/2022.3.62f3/Unity.app/Contents/MacOS/Unity' ./build_desktop_pet.sh
  ```
  跨平台打包需先在 Unity Hub 装好对应 Build Support 模块。

脚本会强制设置：`Fullscreen Mode = Windowed`、`Run In Background`、Windows 用 `Direct3D11` + flip-model 交换链、去开屏 Logo，并把 CatWidget 场景加入构建列表。

### 为什么之前是黑底（已修复）
1. Windows 上用了 `WS_EX_LAYERED` 分层窗口，它和 flip-model 交换链不兼容，合成结果是黑屏。已改为纯 DWM 逐像素 alpha，不再用 layered。
2. 相机开着 HDR/MSAA 会丢掉 alpha 通道。`DesktopPet` 现在启动时会自动把相机 `allowHDR/allowMSAA` 关掉。
3. 图形 API 必须是 D3D11（flip-model 透明依赖它）。打包脚本已强制。

## Windows

无需额外插件，`TransparentWindow.cs` 直接用系统 user32/dwmapi。用上面的打包脚本出包即可。

## macOS

macOS 需要一个 Cocoa/Metal 原生插件才能透明+无边框。**打包脚本会自动帮你编译**：
在 **macOS 上**用 `Build > Desktop Pet - macOS`（或 `./build_desktop_pet.sh mac`）时，
会先用 clang 把 `Assets/Plugins/macOS/TransparentWindowMac.mm` 编成通用二进制
`TransparentWindowMac.bundle` 并自动标记为 macOS 插件，再打包。

前置条件：
- **必须在 Mac 上打 macOS 包**（在别的系统上打，插件不会被编译，就会出现黑底+窗体）。
- 需安装 Xcode Command Line Tools（`xcode-select --install`）。

之前黑底+有窗体的原因：`.bundle` 根本没被编译，原生代码没运行。现在脚本会自动编。
若想手动编：
```bash
cd Assets/Plugins/macOS
clang -bundle -arch arm64 -arch x86_64 \
      -framework Cocoa -framework QuartzCore -framework Metal \
      -o TransparentWindowMac.bundle TransparentWindowMac.mm
```

透明实现要点（已在插件里处理）：把窗口设为非不透明并**递归找到 Unity 的 CAMetalLayer 设 opaque=NO**——
这是 macOS 背景不黑的关键。`Metal Framebuffer Only` 工程里已是关闭状态。

- 首次运行系统可能提示权限，允许后置顶/穿透才完全生效。
- 若仍是黑底：运行后看 `~/Library/Logs/...Player.log` 或控制台是否有
  `[TransparentWindow] 未找到...bundle` —— 有则说明插件没打进包，确认在 Mac 上打的包。

## 行为说明

- **点击穿透**：每帧用系统全局光标位置对 UI 做射线检测，命中可交互图形就临时关闭穿透，离开再打开。计时条设为不挡射线，可直接穿透。
- **拖拽**：`CatDragHandler` 在超过拖拽阈值后移动小猫，并把落点记为待机基点；因触发了拖拽，EventSystem 不会再误发单击。
- **置顶**：`DesktopPet.alwaysOnTop`（Inspector 可关）。

## 涉及文件

- `Assets/Scripts/UIWidget/TransparentWindow.cs` —— 跨平台窗口层（透明/无边框/置顶/穿透/全局光标）
- `Assets/Scripts/UIWidget/DesktopPet.cs` —— 挂在 CatWidget 上，透明相机 + 穿透切换
- `Assets/Scripts/UIWidget/CatDragHandler.cs` —— 挂在 Cat 上，拖拽移动
- `Assets/Plugins/macOS/TransparentWindowMac.mm` —— macOS 原生插件源码
